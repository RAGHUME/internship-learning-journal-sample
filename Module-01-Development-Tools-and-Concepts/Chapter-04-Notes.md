# Chapter 04 – Code Editor, VS Code & Productive Workflow (Week 1 – Session 4)

## 1. Purpose of This Chapter
This chapter introduces **Visual Studio Code (VS Code)** and explains how a modern code editor improves productivity in **data science and software development workflows**.

The focus is on:
- Setting up VS Code
- Understanding editor features
- Integrating terminal, Git, and extensions

---

## 2. What is a Code Editor?
A **code editor** is a tool used to write, edit, and manage source code efficiently.

### Why Not Use Basic Editors?
Basic editors (like Notepad) lack:
- Syntax highlighting
- Error detection
- Debugging support
- Version control integration

---

## 3. Introduction to Visual Studio Code (VS Code)

### VS Code Features:
- Free and open source
- Lightweight and fast
- Supports multiple programming languages
- Large extension ecosystem
- Built-in terminal and Git support

---

## 4. Installing VS Code
VS Code can be installed on:
- Windows
- macOS
- Linux

After installation:
- Launch VS Code
- Set default theme and preferences
- Enable auto-save (recommended)

---

## 5. VS Code Interface Overview

### Main Components:
| Component | Description |
|--------|------------|
| Activity Bar | Access Explorer, Git, Extensions |
| Explorer | View files and folders |
| Editor | Write and edit code |
| Status Bar | Displays file info, errors |
| Terminal | Run commands inside VS Code |
