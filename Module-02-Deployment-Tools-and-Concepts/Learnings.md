# 📘 Week 2 – Deployment Tools & Concepts

## 🔹 1. Docker and Containerization

Docker is a platform used to develop, ship, and run applications inside containers.

### Key Concepts:
- Container → Lightweight runtime environment  
- Image → Blueprint of a container  
- Docker Hub → Public registry for images  

### Advantages:
- Platform-independent  
- Faster deployment  
- Isolated environments  

---

## 🔹 2. Docker Architecture

Docker follows a client-server architecture:
- Docker Client → Runs commands  
- Docker Daemon → Executes containers  
- Docker Registry → Stores images  

---

## 🔹 3. Container Lifecycle

1. Pull image  
2. Run container  
3. Monitor  
4. Stop/remove  

---

## 🔹 4. Docker Networking

Docker provides networking to allow communication between containers.

- Bridge network (default)
- Custom networks for microservices

---

## 🔹 5. Volume Mounting

Used to persist data outside containers.

### Benefits:
- Data persistence  
- File sharing  
- Backup support  

---

## 🔹 6. Running AI Models in Containers

Docker can be used to run LLMs like Ollama.

### Benefits:
- Easy setup  
- Isolation  
- Scalable deployment
